PersonaMimic Production Style Guide1. Architectural Philosophy (Studio Standard)Domain-Driven Design (DDD): Group code by feature domain (e.g., app/users/, app/payments/) rather than technical type. Each domain must have its own models.py, schemas.py, repository.py, and service.py .Service-Repository Pattern: Strictly decouple logic. Route handlers are only for orchestration. Business logic lives in service.py. Database/IO access lives in repository.py .Composition Over Inheritance: Favor composition to share functionality.Dependency Injection: Use FastAPI dependencies for database sessions, auth, and external clients .2. Structural Simplification & LogicGuard Clauses: Always replace nested "pyramid" logic with early returns. Reject invalid states at the top of functions to keep the "happy path" at the root level of indentation .Pattern Matching: Use match/case (Python 3.10+) for complex structural branching and variable binding .Atomic Decomposition: If a function exceeds 40 lines, refactor it into small, named helper steps .Signature Limits: Maximum 3-5 arguments per function. Group larger sets into dataclasses or TypedDicts .3. Formatting & NamingPEP 8+: Adhere to PEP 8, but extend physical line limits to 100 characters for efficiency.Imports: Use absolute imports. Group into: (1) Standard Library, (2) Third-party, (3) Local modules.Naming: Use snake_case for variables/functions and PascalCase for classes. Use __ for throwaway variables to avoid conflicts with gettext .No Magic Values: Extract all literal numbers or strings to named constants or Enum types .4. Modern Python & Performance (2025-2026)T-Strings (Python 3.14+): Use t"..." for safe SQL and HTML generation to prevent injection .F-Strings: Use 3.12+ features (quote reuse and inline comments) for complex formatting .Iterator Algebra: Prefer itertools (e.g., batched, chain) and collections.defaultdict for data processing. Aim for $O(n)$ performance and memory-efficient lazy evaluation .Structured Logging: Use JSON-based structured logging with key-value context for all production logs .5. Concurrency & Error HandlingStructured Concurrency: Use asyncio.TaskGroup (3.11+) for all parallel I/O .Safe Blocking: Wrap unavoidable blocking synchronous calls in run_in_executor.Exception Specificity: Catch specific exceptions. Never use bare except: or except Exception:. Log context for all failures .Free-Threading (3.14+): Identify CPU-bound parallel workloads to utilize the no-GIL (free-threaded) runtime .6. Tooling & VerificationToolchain: Exclusively use uv for dependency management and ruff for all linting and formatting .Test-First Delivery: Every feature must include a mirrored test_*.py file using pytest .Artifact Integrity: Before final submission, run ruff check. and ensure the implementation_plan.md artifact reflects the final architecture.[20, 22]
